              Create household

              Add another isolated store. Requires the server master password.

              Server tools for the default household: read-only overview, on-demand JSON backups under the data directory, and Web Push (VAPID) keys.

              Server overview

              Non-secret flags and household ids registered on this process. Values come from environment variables and the household registry.

              On-demand backup

              Writes a JSON snapshot of this household to data/households/<id>/backups/ (same format as automatic backups before import). Old files are pruned using CHORELOG_BACKUP_RETENTION.

              Web Push (VAPID) keys

              VAPID keys for Web Push (PWA). Values are saved to the server data directory so they survive restarts when environment variables are not set.

              Use a real mailto: or https: URL. Some push services reject placeholder contacts.

              Audit log

              Changes to the activity log and household settings (newest first). The signed-in account name is stored as the actor.

                Data

                Import can replace everything or merge with a prompt. JSON export includes people, log entries, and scheduled chores. CSV is log entries only (spreadsheet-friendly).

                Scheduled chores
